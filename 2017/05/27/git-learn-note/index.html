git学习笔记

发表于 2017-05-27 | |
| 字数统计 607 | 阅读时长 2

在learn git branching上进行学习，记录一些命令。

  1. 提交操作，$ git commit对当前分支进行提交，产生新的记录并记录下其父节点
  2. 创建和切换分支，$ git branch <分支名>创建一个新的分支，$ git checkout <分支名>切换到某分支，git checkout -b <分支名>这个命令是个组合命令，可以直接创建并切换到该分支
  3. 分支与合并$ git merge <分支名>，将某一分支合并到当前分支，被合并的分支提交记录将在当前分支得以体现。
  4. 线性分支合并，$ git rebase <分支名>将当前分支的所在的记录链复制一份移到目标分支，如果有相同提交记录则合并。
  5. 使用-f 选项让分支指向另一个提交，例如：$git branch -f master HEAD~3,将master分支强制移动到HEAD的3级父提交。
  6. 撤销变更，在当前提交记录基础上撤销，本地撤销使用$ git reset HEAD~1向前回退一次提交，但是这只是在本地更改，被撤销的操作是未被存入暂存区，但是已提交到远成仓库的不受影响，远程仓库回退用$ git revert HEAD就是相当于在远程仓库提交一个上一版本的记录。
  7. $ git cherry-pick <分支号哈希值> <分支号哈希值>...分支号是哈希值，不是分支名，将目标几个提交记录提交到当前分支下
  8. 交互式rebase，指的是–interactive的rebase命令，简写为-i,在其后加上HEAD~n就是向前几个节点，例如$ git rebase -i HEAD~4,之后在弹出的UI页面进行拖动和对pick状态进行更改，即可实现调整记录顺序。
  9. $ git commit --amend提交并移动分支
  10. git的tag标记，$ git tag <标记名> <分支哈希值>，该tag是该提交记录绑定的。
  11. $ git describe <ref>用来描述离你最近的锚点，也就是tag，可以是任何能被Git识别成提交记录的引用，没有时默认是HEAD坐在提交记录，其输出结果格式为 <tag>_<numCommits>_g<hash>，tag是离ref最近的标签，numCommits是表示tag与ref相差多少个提交记录，hash表示的是给定的ref的哈希值的前几位。$ git bisect。
  12. ‘^’符号，可以指定选择第几个父提交，
